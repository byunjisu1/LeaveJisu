# ✈️ LeaveJisu

#### $\color{#007AFF}{\Large {1.\ 프로젝트\ 제목}}$ 
#### &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;떠나지수(Leave, Jisu) <br><br>

#### $\color{#007AFF}{\Large {2.\ 프로젝트\ 개발\ 기간}}$
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2025.10 ~ 2026.01 <br><br>

#### $\color{#007AFF}{\Large {3.\ 프로젝트\ 개요}}$
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;여행 계획의 번거로움을 줄이고 ‘즐거운 준비’를 돕기 위해 기획한 여행 정보 플랫폼 <br><br>
  
#### $\color{#007AFF}{\Large {4.\ 프로젝트\ 선정\ 이유}}$
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;여행과 힐링에 대한 관심이 높아진 요즘, 많은 사람들이 시간적 여유 부족으로 여행을 자주 즐기기 어렵다고 느낍니다. 하지만 기존 여행 계획 서비스는 계획 과정이 복잡해서 비효율적이고 지루하게 느껴지는 경우가 많았습니다. 이에 여행 계획을 보다 간편하고 재미있게 세울 수 있는 서비스를 만들고자 여행 정보 플랫폼 Triple을 벤치마킹하게 되었습니다.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Triple은 유용한 기능을 제공하지만, 핵심적인 여행 계획 기능이 모바일 앱에 한정되어 있다는 아쉬움이 있었습니다. 저는 웹의 넓은 화면에서 여행 일정을 한눈에 확인하며 계획할 수 있는 환경이 더 효율적이라고 판단했고, 이를 웹 서비스로 확장하는 프로젝트를 기획했습니다. 또한 AI 기능을 활용해 짐 싸기 등 번거로운 준비 과정을 자동화하여, 사용자가 쉽고 간편하게 여행을 준비할 수 있도록 구현하고자 했습니다. <br><br>

#### $\color{#007AFF}{\Large {5.\ 특장점}}$
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;① AI 추천 : 옵션 선택에 따른 추천 및 사용자의 장소 선택 등 클릭 이력 기반 추천 (ChatGPT API)
  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;② AI 사진 분석 : 짐 싸기 (캐리어 사진을 업로드 하면, 챙겨야 할 물품 리스트를 출력) (ChatGPT API)

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;③ 여행 계획에 담기 : AI 추천에 따른 일정을 내 여행 계획에 담고 수정/보완할 수 있음

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;④ 항공편 : 실시간 항공권 가격 조회로 최저가 검색 (Amadeus API)

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;⑤ 소셜로그인 (Naver API)

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;⑥ 일정별 여행 계획을 지도에서 확인하고 계획하기 (Google Maps API)

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;⑦ 메일 인증 : 회원가입 및 비밀번호 초기화 (JavaMail API) <br><br>

#### $\color{#007AFF}{\Large {6.\ 맡은\ 역할}}$
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;일부 화면(HTML/CSS)을 제외한 모든 화면 및 기능 전부 <br><br>

#### $\color{#007AFF}{\Large {7.\ 사용\ 기술}}$
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Java, Oracle 11.2, HTML/CSS, JS/jQuery, Servlet&JSP, Spring Framework, MyBatis, AJAX, ChatGPT API, Naver Login API, Amadeus API, Google Maps API, JavaMail API <br><br>

#### $\color{#007AFF}{\Large {8.\ 성장점\ 및\ 느낀점}}$

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;① 사용자의 행동 흐름에 따라 로직을 설계하는 것이 서비스의 본질적인 가치를 결정한다는 것을 깊이 깨달음.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;② 어떠한 기능을 구현하더라도 개발자의 편의가 아닌 사용자의 입장에서 먼저 생각해야 하며, 그들의 불편함을 해소하기 위해 최적화된 코드를 고민하는 과정이 필요함을 느꼈음.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;③ 팀원의 빈 자리를 채우며 남겨진 코드를 분석했던 경험을 통해, 시간이 흘러도 의도가 명확히 전달될 수 있도록 주석을 남기고 가독성 높은 코드를 작성하는 습관이 협업과 유지보수에 얼마나 중요한 것인지 배울 수 있었음.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;④ end-to-end로 프로젝트 개발을 경험한 후 웹 서비스를 보는 시각이 달라졌음. 처음에는 화면이 전부인 줄 알았고 뭐부터 만들어야 할지 뭘 만들어야 할지 전혀 몰랐는데, 이제는 주제가 주어지면 기획부터 시작해서 구현 배포까지 완료할 수 있게 되었음.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;⑤ 스프링 프로젝트의 구조를 알고 어떻게 개발해야 하는지 알게 되었음. 이제는 어려움 없이 스프링 레거시 프로젝트(그리고 스프링 부트 프로젝트까지도)를 프로젝트 생성부터 서버 배포까지, 스스로 만들 수 있게 되었음.
